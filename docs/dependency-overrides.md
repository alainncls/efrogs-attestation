# Dependency Overrides

Last reviewed: 2026-05-08.

This pnpm workspace keeps overrides only for transitive dependency security fixes that cannot be reached through direct
package updates yet. Direct dependencies should be preferred over overrides whenever upstream packages release compatible
versions.

## Known Blocked Items

`uuid` remains on vulnerable transitive versions through Reown/Coinbase and Wagmi/Metamask paths. The patched version is
`uuid >=14`, which is a transitive major upgrade and is not forced through an override.

`elliptic` remains reported through the Hardhat verify Ethers v5 stack. The audit feed reports no patched version, so
this is tracked as upstream-only risk in
[#39](https://github.com/alainncls/efrogs-attestation/issues/39).

`wagmi` and `@wagmi/core` are intentionally held on the current compatible line because `@reown/appkit-adapter-wagmi`
still resolves `@wagmi/connectors@7.0.5`; forcing the newer Wagmi 3.6 line creates a peer dependency mismatch. This is
tracked in [#41](https://github.com/alainncls/efrogs-attestation/issues/41).

`@types/node` remains on the Node 24 line because `.nvmrc` and `package.json` still target Node 24. A Node 25 runtime and
type migration is tracked in [#40](https://github.com/alainncls/efrogs-attestation/issues/40).
